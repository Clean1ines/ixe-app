# Вклад в проект ixe-app

Спасибо, что интересуетесь вкладом в `ixe-app`!

## Стиль написания кода

*   Следуйте PEP 8.
*   Используйте type hints.
*   Пишите документацию (docstrings) для публичных функций/классов.

## Процесс внесения изменений

1.  Сделайте fork репозитория.
2.  Создайте ветку `feature/ВАША_ФИЧА` или `bugfix/ИСПРАВЛЕНИЕ_ОШИБКИ`.
3.  Внесите изменения.
4.  Напишите/обновите тесты.
5.  Убедитесь, что все тесты проходят: `pytest`.
6.  Сделайте commit с осмысленным сообщением.
7.  Отправьте pull request в ветку `main`.

## Архитектурные принципы

При внесении изменений придерживайтесь принципов SOLID, DDD и TDD, описанных в README.
